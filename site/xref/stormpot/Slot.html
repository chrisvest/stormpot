16  package stormpot;
17  
18  /**
19   * A Slot represents a location in a Pool where objects can be allocated into,
20   * so that they can be claimed and released. The Slot is used by the allocated
21   * Poolables as a call-back, to tell the pool when an object is released.
22   *
23   * The individual pool implementations provide their own Slot implementations.
24   *
25   * Slots contain mutable state within them. If objects are claimed, released
26   * and otherwise worked on from different threads, then the hand-over must
27   * happen through a safe publication mechanism.
28   *
29   * See Java Concurrency in Practice by Brian Goetz for more information on safe
30   * publication.
31   *
32   * Slots also expects single-threaded access, so they cannot be used
33   * concurrently by multiple threads. Only one thread at a time can use a Slot
34   * instance.
35   * 
36   * @author Chris Vest <mr.chrisvest@gmail.com>
37   * @see Poolable
38   */
39  public interface Slot {
40    /**
41     * Signal to the pool that the currently claimed object in this slot has been
42     * released.
43     *
44     * It is a user error to release a slot that is not currently claimed. It is
45     * likewise a user error to release a Slot while inside the Allocators
46     * {@link Allocator#allocate(Slot) allocate} method, or the Expirations
47     * {@link Expiration#hasExpired(SlotInfo) hasExpired} method.
48     *
49     * On the other hand, it is _not_ an error to release a Poolable
50     * from a thread other than the one that claimed it.
51     *
52     * Pools are free to throw a PoolException if they detect any of these
53     * wrong uses, but it is not guaranteed and the exact behaviour is not
54     * specified. "Unspecified behaviour" means that dead-locks and infinite
55     * loops are fair responses as well. Therefore, heed the advice and don't
56     * misuse release!
57     *
58     * Pools must, however, guarantee that an object is never
59     * {@link Allocator#deallocate(Poolable) deallocated} more than once.
60     * This guarantee must hold even if release is misused.
61     * @param obj A reference to the Poolable object that is allocated for this
62     * slot, and is being released.
63     */
64    void release(Poolable obj);
65  
66    /**
67     * Mark the object represented by this slot as expired. Expired objects
68     * cannot be claimed again, and will be deallocated by the pool at the
69     * earliest convenience. Objects are normally expired automatically using
70     * the {@link Expiration} policy that the pool has been configured with. If
71     * an object through its use is found to be unusable, then it can be
72     * explicitly expired using this method. Objects can only be explicitly
73     * expired while they are claimed, and such expired objects will not be
74     * deallocated until they are released back to the pool.
75     *
76     * It is a user error to expire objects that are not currently claimed. It is
77     * likewise a user error to expire a Slot while inside the Allocators
78     * {@link Allocator#allocate(Slot) allocate} method.
79     *
80     * The expiration only takes effect after the object has been released, so
81     * calling this method from within the Expirations
82     * {@link Expiration#hasExpired(SlotInfo) hasExpired} method will not prevent
83     * the object from being claimed, but will prevent it from being claimed
84     * again after it's been released back to the pool. An Expiration policy that
85     * expires all objects with this method, is effectively allowed objects to
86     * only be claimed once.
87     *
88     * Pools are free to throw a PoolException if they detect any wrong uses,
89     * but it is not guaranteed and the exact behaviour is not specified.
90     * "Unspecified behaviour" means that dead-locks and infinite loops are fair
91     * responses as well. Therefore, heed the advice and don't misuse expire!
92     *
93     * Pools must, however, guarantee that an object is never
94     * {@link Allocator#deallocate(Poolable) deallocated} more than once.
95     * This guarantee must hold even if expire is misused.
96     * @param obj A reference to the Poolable object that is allocated for this
97     * slot, and is being expired.
98     */
99    void expire(Poolable obj);
100 }
